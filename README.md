🛠 TireVibe 개발 지시 프롬프트

Mission:
Flappy Bird 스타일의 캐주얼 리듬+액션 아케이드 게임 TireVibe를 웹에서 구현한다.
플레이어는 찢어진 타이어 밈 캐릭터를 굴리며 장애물을 회피하고, 웹소켓을 통한 멀티플레이/관전자 개입이 가능해야 한다.

1. 기술 스택 및 환경 제약

프론트엔드: 정적 파일(index.html, app.js) + CDN(React, ReactDOM, Tailwind, Babel-standalone)

게임 루프: Canvas API 또는 WebGL (React는 HUD/UI에만 사용)

백엔드: FastAPI (127.0.0.1:8000), 웹소켓 엔드포인트 제공, 정적 파일 서빙

네트워킹: 웹소켓 기반, CORS 허용, 포트는 8000만 사용

설치 정책: npm/pnpm/yarn, 번들러, 패키지 설치 금지 → 반드시 CDN + ESM 구조만 사용

2. 게임 플레이 요구사항

타이어는 자동으로 굴러간다 (X축 이동)

플레이어 입력: 점프(스페이스/탭), 틈새 돌파(Shift/슬라이드)

장애물은 랜덤 생성되며 점차 난이도 상승

피곤 수치 게이지: 시간이 지날수록 제어 난이도가 증가

음악 비트 싱크: 성공 시 표정 변화/밈 대사 출력

실패 시: “뚝” 하고 찢어진 타이어 연출

3. 멀티플레이 & 관전자 기능

멀티플레이: 여러 명의 플레이어 상태(위치, 점프 여부) 서버에서 브로드캐스트

관전자 이벤트:

이모지 입력 → 캐릭터 표정 변화 (😂 → 웃음 표정)

명령어 /shake, /dark → 화면 흔들림, 배경 어두워짐

Vibe-coding 패널: 오른쪽에 미니 코드 입력창 → 즉시 효과 반영

4. 밈 확산 기능

실패 순간 자동 스크린샷 버튼 → SNS 공유

사용자 제작 타이어 얼굴 스킨 등록 가능

5. 구현 단계

FastAPI 서버

정적 파일 서빙 (index.html, app.js)

/ws 웹소켓 엔드포인트 (브로드캐스트 + 관전자 이벤트 반영)

클라이언트 구조

index.html: UI 구조 + Tailwind/React/Canvas

app.js:

Canvas 루프 (requestAnimationFrame)

타이어 이동, 점프, 충돌 판정

웹소켓 연결 → 입력/상태 전송, 이벤트 반영

UI(HUD)

React로 게임 시작 버튼, 점수 표시, 관전자 패널 구현

Canvas는 게임 본체만 담당

멀티플레이/관전자

서버: 틱마다 상태 브로드캐스트

클라이언트: 다른 플레이어 타이어도 렌더링

관전자 메시지 → 타이어 표정/장애물 변형

6. 개발 지시 포맷

코드는 순수 ESM + CDN 구조로 작성 (npm/webpack 금지)

index.html + app.js 기반으로 빠른 바이브 코딩 가능하도록 구성

서버/클라 각각 최소 실행 가능한 버전(MVP) 먼저 완성 후 → 기능 확장

👉 에이전트에게 전달할 실제 프롬프트:

당신은 TireVibe(타이어바이브)라는 캐주얼 리듬 액션 아케이드 게임을 개발하는 게임 클라이언트/서버 개발자다. 
다음 요구사항을 반드시 준수하라:

1. 프론트: index.html + app.js 정적 파일, React/Tailwind/CDN만 사용, Canvas로 게임 루프 구현.
2. 백엔드: FastAPI, 정적 파일 서빙, /ws 웹소켓 엔드포인트 구현.
3. 게임 로직: 타이어 자동 전진 + 점프/틈새 돌파, 장애물 충돌 판정, 피곤 수치, 음악 비트에 반응하는 표정/밈 출력.
4. 네트워킹: 웹소켓으로 플레이어 상태와 관전자 입력을 실시간 반영. 관전자는 이모지/명령어로 게임에 개입할 수 있음.
5. 멀티플레이: 여러 타이어가 동시에 굴러가며 최후의 생존자가 승리.
6. SNS 공유: 실패 순간 스크린샷 캡처 버튼.
7. 모든 코드는 번들링/패키지 설치 없이 즉시 실행 가능해야 함.

최소 실행 가능한 프로토타입(MVP)을 먼저 구현하고, 이후 기능을 점진적으로 확장하라.
